# 从网上导入历年试卷（真实数据来源）

本功能**不会**自动爬取公开网站的整套真题（涉及版权、robots 与站点条款）。要在本地检索并导入「真实」试卷，您需要在**依法有权使用**的前提下，自行准备**试卷目录清单**，由知学按清单抓取正文后再做 AI 结构化入库。

## 数据来源可以是什么

- 贵校 / 机构内部存档（纸质扫描后经 OCR、人工校对得到的纯文本）。
- 您已获得书面授权或适用合理使用范围的教辅与题库导出文本。
- 您自建或购买的合规题库接口：由运维定期产出与本仓库约定格式的 JSON 清单，通过 HTTPS 提供。

请自行确认著作权与使用许可；滥用造成的法律责任与本开源示例无关。

## 清单如何配置

1. **本地文件**：项目目录 `data/remote-paper-catalog.json`，字段见下方。
2. **远程合并**：环境变量 `MPG_REMOTE_IMPORT_CATALOG_URL` 指向 **HTTPS** JSON，结构与本地文件相同；会与本地文件按条目 `id` 合并（后者覆盖同名 id）。

服务端每隔短时间会缓存合并结果；修改本地文件后稍候再检索或重启开发服务。

## JSON 条目字段（每条对应一份试卷）

| 字段 | 说明 |
|------|------|
| `id` | 全局唯一字符串 |
| `year` | 四位年份数字 |
| `gradeId` | 与命题页一致，如 `pri_g3_s1`（三年级上） |
| `subjectId` | 如 `math`、`chinese` |
| `paper_kind` | 可选；与命题页「试卷场景」id 一致，如 `regular_final` |
| `title` | 列表展示标题 |
| `plainText` | 若填写：直接使用该字符串作为卷面正文参与 AI 整理（须有足够长度） |
| `textUrl` | 若不写 `plainText`：须为可 GET 的 URL，返回 **UTF-8 纯文本**（非 PDF；PDF 需先在馆外转为文本） |

每条至少提供 `plainText` 或 `textUrl` 之一。

## 自动化流水线建议（机构）

- OCR / 排版脚本产出正文 → 写入对象存储或静态站点 → 清单里填 `textUrl`。
- CI 将清单发布到内网或受控 CDN → 设置 `MPG_REMOTE_IMPORT_CATALOG_URL` → 应用内按年份、年级等筛选后进入导入队列。

结构示例见 `examples/v1/remote-paper-catalog.sample.json`（条目仅为字段说明，非真题内容）。
