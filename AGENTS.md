# Agent 提示（本仓库）

- **流程与闸门**：命题与发布闭环见 `docs/workflow.md`；人工核对清单见 `docs/validation-checklist.md`。
- **数据结构**：`schemas/v1/*.schema.json`；示例见 `examples/v1/` 与 `papers/2026/demo-2026-amc-style-01/`。
- **校验**：修改 JSON 后运行 `make validate`。
- **前端与服务端（TanStack Start）**：仓库根 `npm install && npm run dev`（workspace：`apps/web`，包名 `@zhixue/web`）；首页 `/`，试卷库 `/library`，生成 `/generate`，演示详情 `/exam/demo`（MPG JSON），`/dashboard` 重定向到 `/`，Schema 快照 `/preview`。
- **桌面应用（Electron）**：`npm run build && npm run desktop` 在本地以窗口打开（先构建再启 `vite preview`）。发布安装包：`npm run desktop:dmg`（输出到 `release/`，需本机已装 Node 且能执行 `node`，因预览进程用系统 Node 启动 Vite）。未签名的 .app 若被 macOS 拦截，可右键「打开」。
- **Electron 安装失败（`socket hang up`）**：仓库已含 `.npmrc` 指向 npmmirror 的 `electron_mirror`；若仍失败，删除 `node_modules/electron` 后重试 `npm install`，或临时执行 `export ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"` 再安装。
- **`vite preview` / Electron 白屏或 500**：构建后会复制 `dist/server/server.js`（与 TanStack 预览入口名一致）；服务端读写 `data/` 使用 `resolveProjectRoot()`（或环境变量 `MPG_PROJECT_ROOT`），避免 SSR 下 `cwd` 为 `/` 时误写 `/data`。
- **听力 Piper（本地免费 TTS）**：配置 `MPG_PIPER_MODEL` 为 Piper `.onnx` 模型绝对路径；可选 `MPG_PIPER_BIN`（默认在 PATH 中查找 `piper`）。未配置且为 macOS 时仍用 `say`。安装步骤见 `docs/listening-piper-setup.md`。**听力稿格式（v3、`__WORD_GAP__`、题面区）：试卷为 `apps/web/public/audio/<卷ID>/listening-script.md`，同型例题为 `…/examples/listening-script.md`，见同文档第 6 节**。
- **网上导入历年卷**：默认 `data/remote-paper-catalog.json` 可为空；真实试卷须自行维护清单（正文 `plainText` / `textUrl`），可选 `MPG_REMOTE_IMPORT_CATALOG_URL` 拉取合并。合规与字段说明见 `docs/remote-paper-catalog.md`。
- **教材章节目录（出版社分册）**：命题页章节可选数据源来自 MySQL 表 `curriculum_catalog_*`；定时由运维执行 `npm run curriculum-catalog:import`（或 cron 调用 `node scripts/import-curriculum-catalog.mjs`），从 `MPG_CURRICULUM_CATALOG_URL` 或 `data/curriculum-catalog.json` 写入。字段与 Cron 示例见 `docs/curriculum-catalog-import.md`。
- **Open Notebook / 正文增强（可选）**：线下导入流程见导入页提示；松耦合 Open Notebook 与 `MPG_PLAINTEXT_EXTRACT_URL` 说明见 `docs/architecture/open-notebook-and-extract-integration.md`。
- **微服务与编排（可选）**：`npm run docker:stack` 拉起网关 + Web + OCR/公式/视觉/Agent Stub；说明见 `docs/architecture/stack-docker.md`。K8s 示例见 `infrastructure/kubernetes/README.md`。
- **可插拔 OCR（结构化 JSON）**：网关 `POST /api/v1/ocr/image` 全量 JSON → `runPluggableOcrPipeline`；目录与 Redis/PG/Agent 约定见 `docs/architecture/ocr-pluggable.md`。兼容入口：`gatewayOcrImage`（纯文本聚合）仍保留。
- **本机开发（Vite :8080 + OCR 网关）**：`npm run dev:host`（`docker:api:detach` 仅 API 栈，不构建 web 镜像）；浏览器 `http://localhost:8080`，`.env` 或进程内 `MPG_GATEWAY_URL=http://127.0.0.1:8080`（Vite 代理 `/api/v1`→8090）。线下导入预览区展示 **OCR 识别来源**（网关 / 浏览器回退）。可选卷修复见 `apps/web/scripts/repair/README.md`；共图大题对齐亦在入库前 `sanitizeImportedSnapshotForPersist` 中执行。
- **教育语义修复**：规则词典 `educationSymbolLexicon.ts`；可选 AI 修复 `services/ocr-ai-repair.server.ts`、`repairOfflineOcrTextWithAi`；线下导入对话框勾选「入库前 AI 语义修复」。本地专用纠错模型键：`localSubjectModels.ocr_repair`（与命题模型分离）。
- **本地 OCR（仅 GOT-OCR 2.0）**：Docker `ocr-service` ← `services/got-ocr-service`（HF `stepfun-ai/GOT-OCR-2.0-hf`，[GOT-OCR2.0](https://github.com/Ucas-HaoranWei/GOT-OCR2.0)）。**离线权重**：`npm run got-ocr:download-model` → `data/hf-models/GOT-OCR-2.0-hf/` 挂载进容器，日常无需联网。**站点加载即自动预热**（`GatewayOcrWarmupRunner`）。`diagram_links` 为右栏启发式。见 `docs/architecture/local-ocr-optimization.md`。
- **几何示意图规则入库**：题干矢量图规则链（尺规复制角 → 旋转落边 → 正方形链）、槽位旋转、数值约束与矩形 PMDN、以及与 OCR/语义修复的并列说明见 `docs/architecture/geometry-diagram-inference-rules.md`。
- **Diagram IR 漂移基准（P6-4）**：corpus 在 `apps/web/tests/fixtures/diagram-ir/corpus/`（`diagram_schema` + `expected.ir` + `expected.diagnostics` + `expected.svg`）。更新 baseline：`npm run diagram-ir:corpus:baseline -w @zhixue/web`；聚合对拍（失败非 0 即漂移）：`npm run diagram-ir:bench -w @zhixue/web`；CI 门禁见 `apps/web/src/lib/diagramIrNormalizeBench.shared.test.ts`。
- **试卷/图片通用规则（项目级，必守）**：试卷结构、图片 URL、图① 序号/数量**不得写死单卷单题**；须按题型与题干/图池推导。规范全文见 `docs/governance/generic-exam-content-policy.md`；导入细则见 `docs/governance/import-generic-rules-policy.md`。单卷差异只进 `data/ocr-repair-lexicon.json` 等可配置层，不进 `src/lib` 专规。**新增功能提交前**：`npm run governance:generic-exam-content`。
- **导入图链 governance（OCR frontend certification）**：ontology / bench / dual-run / CI 宪法见 `docs/governance/`（RFC-0001–0004）；corpus 在 `apps/web/tests/fixtures/import-pipeline/corpus/`。验证：`npm run import-pipeline:bench -w @zhixue/web`、`npm run import-pipeline:dual-run -w @zhixue/web`。Canonical frontend ≠ 最好；experimental 仅 observational。
- **ECR 运行时宪法**：`docs/governance/ECR-RUNTIME-CONSTITUTION-v1.md` — 六平面 frozen truth · 三轴治理（Authority / Fidelity / Temporal）。**Authority** blocking：`governance:projection-purity`；**Fidelity** observational：`projection-fidelity:snapshot|compare`；P3.3 primitives：`projectionPrimitivesContract.shared.ts`（realization substrate，零 authority）。
- **Semantic lineage / epistemic governance（CLI + CI）**：`npm run inspect:semantic-lineage -w @zhixue/web`（`--corpus` = CI frozen corpus；`--scan-local` = 本机卷库；`--gate-*` 默认 `strict`）。周对比：`npm run semantic-telemetry:snapshot` / `semantic-telemetry:compare`（只 diff `data/telemetry-snapshots/`，禁止重跑旧卷）。CI：`.github/workflows/semantic-lineage-governance.yml`（Phase A Coverage + Phase B Quality）。**Foundational invariant**：`UNOBSERVABLE` 不得静默变绿。全文：`docs/governance/SEMANTIC-REPLAY-LINEAGE-v1.md`。
- **卷面图 ownership（P7-1A）**：导入入库前 `sanitizeImportedSnapshotForPersist` 内会执行 `applyImportedExamFigureOwnershipFromRaster`：从 `raster_figures.stem` 建 `exam.figure_registry`、写 `question.figure_refs`，子题（题干以单位数「(1)」等开头）**全量继承**上一道有 stem 图的父题 refs；读卷题干裁图经 `resolveFigureResources(question, exam)`。DB：`sql/mysql/zhixue_schema.sql` 与 `supabase/migrations/20260514120000_exams_figure_registry_questions_figure_refs.sql`；单测 `figureOwnershipApply.shared.test.ts`。调试：试卷页 `?figures_debug=1`（或本地 `import.meta.env.DEV`）展开每题「Figure ownership（调试）」面板。
